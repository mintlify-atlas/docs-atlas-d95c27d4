This guide will walk you through setting up Auditoriapp locally, logging in, creating your first budget period, and submitting your first project for approval.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Python 3.8+** and pip
- **Node.js 16+** and npm
- **PostgreSQL** or **MySQL** (or use SQLite for testing)
- **Git**

## Setup Backend

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/cristiangodoyangel/auditoriapp.git
    cd auditoriapp/backend
    ```
  </Step>

  <Step title="Create virtual environment">
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install -r requirements.txt
    ```
  </Step>

  <Step title="Configure environment">
    Create a `.env` file in the `backend/` directory:

    ```bash .env
    SECRET_KEY=your-secret-key-here
    DEBUG=True
    ALLOWED_HOSTS=*
    
    DB_NAME=auditoria_db
    DB_USER=admin
    DB_PASSWORD=password
    DB_HOST=localhost
    
    JWT_ACCESS_TOKEN_LIFETIME=60
    JWT_REFRESH_TOKEN_LIFETIME=1440
    ```
  </Step>

  <Step title="Run migrations">
    ```bash
    python manage.py migrate
    ```
  </Step>

  <Step title="Create admin user">
    ```bash
    python create_admin_user.py
    ```

    This creates an admin user with credentials:
    - **Username**: `admin`
    - **Password**: `admin123`
    - **Role**: `admin`
  </Step>

  <Step title="Start the backend server">
    ```bash
    python manage.py runserver
    ```

    The API will be available at `http://localhost:8000`
  </Step>
</Steps>

## Setup Frontend

<Steps>
  <Step title="Navigate to frontend directory">
    ```bash
    cd ../frontend
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Start development server">
    ```bash
    npm run dev
    ```

    The frontend will be available at `http://localhost:5173`
  </Step>
</Steps>

## First Login

<Steps>
  <Step title="Open the application">
    Navigate to `http://localhost:5173` in your browser.
  </Step>

  <Step title="Log in with admin credentials">
    - **Username**: `admin`
    - **Password**: `admin123`

    The system will issue a JWT access token (valid for 60 minutes) and store it in localStorage.
  </Step>

  <Step title="Verify dashboard access">
    After login, you should see the main dashboard with KPI cards showing:
    - Total budget
    - Total expenses
    - Budget execution percentage
    - Available balance
  </Step>
</Steps>

## Create Your First Period

Budget periods organize funding into discrete time frames with allocated budgets.

<Steps>
  <Step title="Navigate to Periods">
    Click **Periodos** in the sidebar navigation.
  </Step>

  <Step title="Create new period">
    Click **Crear Periodo** and fill in the form:

    - **Name**: "Q1 2024"
    - **Start Date**: 2024-01-01
    - **End Date**: 2024-03-31
    - **Allocated Budget**: 50000000 (50 million Chilean pesos)
    - **Community**: Select your community from the dropdown
  </Step>

  <Step title="Save the period">
    Click **Guardar**. The period will be created and marked as active.
  </Step>
</Steps>

<Note>
  Only one period can be active at a time per community. When you close a period, the remaining balance automatically transfers to the next period.
</Note>

## Create Your First Project

<Steps>
  <Step title="Navigate to Projects">
    Click **Proyectos** in the sidebar.
  </Step>

  <Step title="Click Create Project">
    Click **Crear Proyecto** to open the project creation form.
  </Step>

  <Step title="Fill in basic information">
    - **Project Name**: "Community Center Renovation"
    - **Description**: "Renovate the community center main hall"
    - **Start Date**: 2024-02-01
    - **End Date**: 2024-03-15
    - **Total Budget**: 10000000 (10 million pesos)
    - **Period**: Select "Q1 2024"
  </Step>

  <Step title="Add strategic information">
    - **Objectives**: "Improve community facilities and provide a safe gathering space"
    - **Justification**: "The current hall is deteriorating and poses safety risks"
    - **Estimated Beneficiaries**: 500
  </Step>

  <Step title="Upload required documents">
    Projects require three documents:
    - **Assembly Minutes (Acta)**: Meeting minutes approving the project
    - **Quotations (Cotizaciones)**: At least 3 price quotes from vendors
    - **Selected Quote (Elegido)**: The chosen vendor's final quote

    Upload each document using the file upload fields.
  </Step>

  <Step title="Add governance information">
    - **Assembly Quorum**: 45 (number of members present at approval meeting)
    - **Presidential Approval**: Check the box to indicate the president has signed off
  </Step>

  <Step title="Save the project">
    Click **Guardar**. The project will be created in **Borrador** (Draft) status.
  </Step>
</Steps>

## Submit for Approval

<Steps>
  <Step title="Review the project">
    Verify all information is correct on the project detail page.
  </Step>

  <Step title="Change status to Internal Review">
    Click **Cambiar Estado** and select **En Revisión Interna**.

    This notifies the community directorate (Directorio/Presidente roles) that the project is ready for review.
  </Step>

  <Step title="Community approval">
    Users with **Presidente** or **Directorio** roles can now review and approve the project, moving it to **En Revisión CPA**.
  </Step>

  <Step title="CPA approval">
    A user with the **CPA** role reviews the project and either:
    - Approves it → status changes to **Aprobado**
    - Requests observations → status changes to **Con Observaciones**
  </Step>

  <Step title="Begin execution">
    Once approved, change the status to **En Ejecución** to begin tracking expenses against this project.
  </Step>
</Steps>

## Create Your First Expense Report

<Steps>
  <Step title="Navigate to Rendiciones">
    Click **Rendiciones** in the sidebar.
  </Step>

  <Step title="Create new expense report">
    Click **Crear Rendición** and fill in:

    - **Project**: Select "Community Center Renovation"
    - **Description**: "Paint and materials for main hall"
    - **Amount**: 1500000 (1.5 million pesos)
    - **Document Number**: "FC-12345" (invoice or receipt number)
    - **Date**: 2024-02-15
  </Step>

  <Step title="Attach supporting documents">
    Upload the invoice, receipt, or proof of purchase.
  </Step>

  <Step title="Submit for review">
    The expense report is created with status **Pendiente** (Pending).

    A CPA or Admin can now review and approve/reject it.
  </Step>
</Steps>
