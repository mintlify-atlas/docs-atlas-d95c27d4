### Review Expense Submission

<api method="POST" endpoint="/api/rendiciones/{id}/revisar/" auth="bearer" />

Review an expense submission and update its status. Only CPA and Admin roles can review.

**Authentication Required:** Yes (CPA or Admin role)

<ParamField path="id" type="integer" required>
  Expense submission ID
</ParamField>

<ParamField body="estado" type="string" required>
  New status: `aprobado`, `observado`, or `rechazado`
</ParamField>

<ParamField body="observacion" type="string">
  Reviewer's observations or comments
</ParamField>

<ResponseExample>
```json
{
  "status": "ok",
  "estado": "aprobado"
}
```
</ResponseExample>

<CodeGroup>
```bash cURL
curl -X POST https://api.auditoriapp.com/api/rendiciones/2/revisar/ \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "estado": "aprobado",
    "observacion": "Documentación completa y correcta"
  }'
```

```python Python
import requests

headers = {'Authorization': f'Bearer {access_token}'}

    'estado': 'observado',
    'observacion': 'Falta factura original'
}

    'https://api.auditoriapp.com/api/rendiciones/2/revisar/',
    headers=headers,
    json=data
)

```

## Expense States

| State | Description |
|-------|-------------|
| `pendiente` | Pending review - Initial submission |
| `aprobado` | Approved by CPA/Admin |
| `observado` | Has observations - Requires corrections |
| `rechazado` | Rejected - Not valid |
| `pagado` | Paid - Reimbursement completed |

## Workflow

1. **Submit**: User creates expense submission with `pendiente` status
2. **Review**: CPA/Admin reviews and sets status to `aprobado`, `observado`, or `rechazado`
3. **Payment**: If approved, status can be updated to `pagado` when reimbursement is processed
4. **Corrections**: If `observado`, user can update and resubmit for review

## Financial Reports

Expense submissions marked as `pagado` are included in:
- Period closing calculations
- Dashboard KPIs
- Project total expenditure tracking
