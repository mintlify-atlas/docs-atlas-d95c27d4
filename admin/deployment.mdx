## Pre-Deployment Checklist

Before deploying to production, ensure you have completed the following:

<Steps>
  <Step title="Security Configuration">
    - Set `DEBUG = False` in settings
    - Configure strong `SECRET_KEY`
    - Set proper `ALLOWED_HOSTS`
    - Enable HTTPS settings
    - Configure CORS properly
  </Step>

  <Step title="Database Setup">
    - Set up production database (MySQL/PostgreSQL)
    - Run migrations
    - Create superuser account
    - Backup strategy in place
  </Step>

  <Step title="Static Files">
    - Collect static files
    - Configure static file serving
    - Set up media file storage
  </Step>

  <Step title="Environment Variables">
    - All sensitive data in environment variables
    - Production `.env` file configured
    - No secrets in version control
  </Step>
</Steps>

## Production Settings

### Security Settings

Update `backend/auditoriapp/settings.py` for production:

```python
# Security
DEBUG = False
SECRET_KEY = os.getenv('SECRET_KEY')
ALLOWED_HOSTS = ['yourdomain.com', 'www.yourdomain.com']

# HTTPS
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
CRSF_COOKIE_SECURE = True
SECURE_BROWSER_XSS_FILTER = True
SECURE_CONTENT_TYPE_NOSNIFF = True
X_FRAME_OPTIONS = 'DENY'

# HSTS
SECURE_HSTS_SECONDS = 31536000
SECURE_HSTS_INCLUDE_SUBDOMAINS = True
SECURE_HSTS_PRELOAD = True
```

### CORS Configuration

Replace development CORS settings (`backend/auditoriapp/settings.py:173-180`):

```python
# Remove this in production
# CORS_ALLOW_ALL_ORIGINS = True

# Use specific origins
CORS_ALLOWED_ORIGINS = [
    "https://yourdomain.com",
    "https://www.yourdomain.com",
]
CORS_ALLOW_CREDENTIALS = True
```

### Database Configuration

Configure production database using environment variables:

```python
DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.mysql",
        "NAME": os.getenv("DATABASE_NAME"),
        "USER": os.getenv("DATABASE_USER"),
        "PASSWORD": os.getenv("DATABASE_PASSWORD"),
        "HOST": os.getenv("DATABASE_HOST"),
        "PORT": os.getenv("DATABASE_PORT", "3306"),
        "OPTIONS": {
            "charset": "utf8mb4",
            "init_command": "SET sql_mode='STRICT_TRANS_TABLES'",
        },
    }
}
```

## Deployment Options

### Option 1: Traditional Server (Gunicorn + Nginx)

<Steps>
  <Step title="Install production dependencies">
    Gunicorn is already in requirements.txt (`backend/requirements.txt:55`):
    ```bash
    pip install gunicorn==22.0.0 whitenoise==6.7.0
    ```
  </Step>

  <Step title="Configure Gunicorn">
    Create `gunicorn_config.py`:
    ```python
    bind = "0.0.0.0:8000"
    workers = 4
    worker_class = "sync"
    max_requests = 1000
    timeout = 30
    keepalive = 2
    errorlog = "/var/log/gunicorn/error.log"
    accesslog = "/var/log/gunicorn/access.log"
    loglevel = "info"
    ```
  </Step>

  <Step title="Create systemd service">
    Create `/etc/systemd/system/auditoriapp.service`:
    ```ini
    [Unit]
    Description=Auditoriapp Gunicorn Service
    After=network.target

    [Service]
    User=www-data
    Group=www-data
    WorkingDirectory=/var/www/auditoriapp/backend
    Environment="PATH=/var/www/auditoriapp/venv/bin"
    EnvironmentFile=/var/www/auditoriapp/.env
    ExecStart=/var/www/auditoriapp/venv/bin/gunicorn \
              --config gunicorn_config.py \
              auditoriapp.wsgi:application

    [Install]
    WantedBy=multi-user.target
    ```
  </Step>

  <Step title="Configure Nginx">
    Create `/etc/nginx/sites-available/auditoriapp`:
    ```nginx
    upstream auditoriapp {
        server 127.0.0.1:8000;
    }

    server {
        listen 80;
        server_name yourdomain.com www.yourdomain.com;
        return 301 https://$server_name$request_uri;
    }

    server {
        listen 443 ssl http2;
        server_name yourdomain.com www.yourdomain.com;

        ssl_certificate /etc/ssl/certs/yourdomain.crt;
        ssl_certificate_key /etc/ssl/private/yourdomain.key;

        client_max_body_size 10M;

        location /static/ {
            alias /var/www/auditoriapp/backend/staticfiles/;
        }

        location /media/ {
            alias /var/www/auditoriapp/backend/media/;
        }

        location / {
            proxy_pass http://auditoriapp;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
    ```
  </Step>

  <Step title="Enable and start services">
    ```bash
    sudo systemctl enable auditoriapp
    sudo systemctl start auditoriapp
    sudo systemctl enable nginx
    sudo systemctl restart nginx
    ```
  </Step>
</Steps>

### Option 2: Docker Deployment

Create a `Dockerfile` for the backend:

```dockerfile
FROM python:3.11-slim

ENV PYTHONUNBUFFERED=1

WORKDIR /app

# Install system dependencies
RUN apt-get update && apt-get install -y \
    gcc \
    default-libmysqlclient-dev \
    pkg-config \
    && rm -rf /var/lib/apt/lists/*

# Install Python dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application
COPY . .

# Collect static files
RUN python manage.py collectstatic --noinput

EXPOSE 8000

CMD ["gunicorn", "--bind", "0.0.0.0:8000", "auditoriapp.wsgi:application"]
```

Create a `Dockerfile` for the frontend:

```dockerfile
FROM node:18-alpine AS build

WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

Create `docker-compose.yml`:

```yaml
version: '3.8'

services:
  db:
    image: mysql:8.0
    environment:
      MYSQL_DATABASE: ${DATABASE_NAME}
      MYSQL_USER: ${DATABASE_USER}
      MYSQL_PASSWORD: ${DATABASE_PASSWORD}
      MYSQL_ROOT_PASSWORD: ${DATABASE_ROOT_PASSWORD}
    volumes:
      - mysql_data:/var/lib/mysql
    ports:
      - "3306:3306"

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    environment:
      - DATABASE_HOST=db
      - REDIS_URL=redis://redis:6379/0
    env_file:
      - .env
    volumes:
      - ./backend:/app
      - static_volume:/app/staticfiles
      - media_volume:/app/media
    ports:
      - "8000:8000"
    depends_on:
      - db
      - redis
    command: gunicorn --bind 0.0.0.0:8000 auditoriapp.wsgi:application

  celery:
    build:
      context: ./backend
      dockerfile: Dockerfile
    command: celery -A auditoriapp worker -l info
    env_file:
      - .env
    depends_on:
      - db
      - redis

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  mysql_data:
  static_volume:
  media_volume:
```

Deploy using Docker Compose:

```bash
docker-compose up -d
```

<Tabs>

    ```nginx
    server {
        listen 80;
        server_name yourdomain.com;
        root /var/www/auditoriapp/frontend/dist;
        index index.html;

        location / {
            try_files $uri $uri/ /index.html;
    ```

  <Tab title="S3 + CloudFront">

## Static Files

Auditoriapp uses WhiteNoise for static file serving (`backend/requirements.txt:56`).

### Collect Static Files

```bash
python manage.py collectstatic --noinput
```

Static files are collected to `STATIC_ROOT` (`backend/auditoriapp/settings.py:130`).

### Configure WhiteNoise

Add to middleware in `settings.py`:

```python
MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',
    'whitenoise.middleware.WhiteNoiseMiddleware',  # Add this
    # ... other middleware
]

# WhiteNoise configuration
STATICFILES_STORAGE = 'whitenoise.storage.CompressedManifestStaticFilesStorage'
```

## Backup Strategy

### Database Backups

<Tabs>
  <Tab title="MySQL">
    ```bash
    # Create backup
    mysqldump -u $DB_USER -p$DB_PASSWORD $DB_NAME > backup_$(date +%Y%m%d).sql

    # Restore backup
    mysql -u $DB_USER -p$DB_PASSWORD $DB_NAME < backup_20260314.sql
    ```
  </Tab>

  <Tab title="Automated with cron">
    ```bash
    # Add to crontab
    0 2 * * * /path/to/backup_script.sh
    ```

    Create `backup_script.sh`:
    ```bash
    #!/bin/bash
    BACKUP_DIR="/var/backups/auditoriapp"
    DATE=$(date +%Y%m%d_%H%M%S)
    mysqldump -u $DB_USER -p$DB_PASSWORD $DB_NAME | gzip > $BACKUP_DIR/db_$DATE.sql.gz
    find $BACKUP_DIR -type f -mtime +30 -delete
    ```
  </Tab>
</Tabs>

### Media Files Backup

```bash
# Backup media files
tar -czf media_backup_$(date +%Y%m%d).tar.gz /path/to/media/

# Sync to S3
aws s3 sync /path/to/media/ s3://your-backup-bucket/media/
```

## Health Checks

Create a health check endpoint:

```python
# views.py
from django.http import JsonResponse
from django.db import connection

def health_check(request):
    try:
        with connection.cursor() as cursor:
            cursor.execute("SELECT 1")
        return JsonResponse({"status": "healthy", "database": "connected"})
    except Exception as e:
        return JsonResponse({"status": "unhealthy", "error": str(e)}, status=500)
```

  <Step title="Revert code">

  <Step title="Restore database">

## Troubleshooting

<AccordionGroup>
  <Accordion title="502 Bad Gateway">
    Check if Gunicorn is running:
    ```bash
    sudo systemctl status auditoriapp
    ```
    
    Check Nginx error logs:
    ```bash
    sudo tail -f /var/log/nginx/error.log
    ```
  </Accordion>

  <Accordion title="Static files not loading">
    Ensure static files are collected:
    ```bash
    python manage.py collectstatic --noinput
    ```
    
    Check Nginx static file configuration
  </Accordion>

  <Accordion title="Database connection errors">
    Verify database credentials in environment variables
    
    Check database service status:
    ```bash
    sudo systemctl status mysql
    ```
  </Accordion>

  <Accordion title="CORS errors">
    Verify `CORS_ALLOWED_ORIGINS` includes your frontend domain
    
    Check that `corsheaders` middleware is properly configured
  </Accordion>
</AccordionGroup>
