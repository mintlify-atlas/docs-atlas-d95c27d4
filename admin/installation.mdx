## Prerequisites

Before installing Auditoriapp, ensure you have the following installed:

- Python 3.11 or higher
- Node.js 18.x or higher
- MySQL 8.0 or higher (or SQLite for development)
- pip (Python package manager)
- npm or yarn (Node package manager)

## Backend Installation

### Using pip (Recommended)

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <repository-url>
    cd auditoriapp/source/backend
    ```
  </Step>

  <Step title="Create virtual environment">
    ```bash
    python -m venv venv
    
    # On Windows
    venv\Scripts\activate
    
    # On macOS/Linux
    source venv/bin/activate
    ```
  </Step>

  <Step title="Install Python dependencies">
    ```bash
    pip install -r requirements.txt
    ```

    This will install all required packages including:
    - Django 5.0.7
    - Django REST Framework 3.15.2
    - djangorestframework-simplejwt 5.3.0
    - django-cors-headers 4.3.1
    - drf-spectacular 0.27.2
    - And more (see full list in requirements.txt)
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the backend directory (see [Configuration](/admin/configuration) for details):
    
    ```bash
    SECRET_KEY=your-secret-key-here
    DEBUG=True
    ALLOWED_HOSTS=localhost,127.0.0.1
    DATABASE_ENGINE=django.db.backends.sqlite3
    ```
  </Step>

  <Step title="Run database migrations">
    ```bash
    python manage.py migrate
    ```
  </Step>

  <Step title="Create superuser">
    ```bash
    python manage.py createsuperuser
    ```
    
    Or use the provided script for a default admin user:
    ```bash
    python create_admin_user.py
    ```
    
    Default credentials:
    - Username: `admin`
    - Password: `admin123`
    - Email: `admin@admin.cl`
  </Step>

  <Step title="Collect static files">
    ```bash
    python manage.py collectstatic --noinput
    ```
  </Step>

  <Step title="Start development server">
    ```bash
    python manage.py runserver
    ```
    
    The API will be available at `http://localhost:8000`
  </Step>
</Steps>

## Frontend Installation

### Using npm

<Steps>
  <Step title="Navigate to frontend directory">
    ```bash
    cd ../frontend
    ```
  </Step>

  <Step title="Install Node dependencies">
    ```bash
    npm install
    ```

    This will install:
    - React 19.1.1
    - React Router DOM 7.9.4
    - Tailwind CSS 4.1.17
    - Vite 7.1.7
    - Recharts 3.5.1
    - DaisyUI 5.5.4
  </Step>

  <Step title="Configure API endpoint">
    Update the API base URL in your frontend configuration to point to your backend:
    
    ```javascript
    const API_BASE_URL = 'http://localhost:8000/api';
    ```
  </Step>

  <Step title="Start development server">
    ```bash
    npm run dev
    ```
    
    The frontend will be available at `http://localhost:5173`
  </Step>
</Steps>

## Docker Installation (Alternative)

For a containerized setup, you can use Docker Compose:

```bash
docker-compose up -d
```

<Note>
Docker configuration files should be created based on your deployment requirements.
</Note>

## Verify Installation

<Steps>
  <Step title="Check backend health">
    Visit `http://localhost:8000/admin` to access the Django admin panel
  </Step>

  <Step title="Check API documentation">
    Visit `http://localhost:8000/api/schema/swagger-ui/` for Swagger documentation
  </Step>

  <Step title="Check frontend">
    Visit `http://localhost:5173` to access the React application
  </Step>
</Steps>

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Module import errors">
    Ensure your virtual environment is activated and all dependencies are installed:
    ```bash
    pip install -r requirements.txt
    ```
  </Accordion>

  <Accordion title="Database connection errors">
    For MySQL connection issues, ensure:
    - MySQL server is running
    - Database credentials are correct in your environment variables
    - PyMySQL or mysqlclient is properly installed
  </Accordion>

  <Accordion title="CORS errors">
    Verify that the frontend URL is added to `CORS_ALLOWED_ORIGINS` in settings.py
  </Accordion>

  <Accordion title="Static files not loading">
    Run `python manage.py collectstatic` and ensure `STATIC_ROOT` is configured
  </Accordion>
</AccordionGroup>
