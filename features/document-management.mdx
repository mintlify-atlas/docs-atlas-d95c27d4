## Overview

Auditoriapp includes a comprehensive document management system that handles file uploads, organization, and attachment to projects and expense reports. The system supports various document types and ensures proper categorization for audit purposes.

## Document Model

The core `Documento` model stores all uploaded files:

```python
class Documento(models.Model):
    fecha_subida = models.DateTimeField(auto_now_add=True)
    proyecto = models.ForeignKey('proyectos.Proyecto', on_delete=models.SET_NULL, 
                                  null=True, blank=True)
    nombre = models.CharField(max_length=255)
    tipo = models.CharField(max_length=50)
    archivo = models.FileField(upload_to='documentos/')
    descripcion = models.TextField(blank=True)

    def __str__(self):
        return f"Documento {self.nombre} - {self.tipo}"
```

### Key Fields

<ParamField path="fecha_subida" type="DateTime">
  Automatically set when document is uploaded
</ParamField>

<ParamField path="proyecto" type="ForeignKey">
  Optional link to a specific project. Can be null for general community documents.
</ParamField>

<ParamField path="nombre" type="String">
  User-friendly name for the document
</ParamField>

<ParamField path="tipo" type="String">
  Document category (e.g., "acta", "cotizacion", "factura", "contrato")
</ParamField>

<ParamField path="archivo" type="File">
  The actual uploaded file, stored in `media/documentos/`
</ParamField>

<ParamField path="descripcion" type="Text">
  Optional additional context about the document
</ParamField>

## Relationships

### Project Documents

Documents can be directly linked to projects:

```python
class Proyecto(models.Model):
    # ... other fields
    
    # Access documents via reverse relationship
    # proyecto.documento_set.all()
```

**Query Example:**
```python
# Get all documents for a project
proyecto = Proyecto.objects.get(id=1)
documentos = proyecto.documento_set.all()

# Filter by type
actas = proyecto.documento_set.filter(tipo='acta')
cotizaciones = proyecto.documento_set.filter(tipo='cotizacion')
```

### Expense Report Attachments

Rendiciones use a ManyToMany relationship for multiple attachments:

```python
class Rendicion(models.Model):
    # ... other fields
    documentos_adjuntos = models.ManyToManyField(
        'documentos.Documento', 
        related_name='rendiciones_adjuntas', 
        blank=True
    )
```

**Usage Example:**
```python
# Attach documents to an expense report
rendicion = Rendicion.objects.get(id=1)
factura = Documento.objects.get(id=5)
rendicion.documentos_adjuntos.add(factura)

# Get all documents for a rendicion
docs = rendicion.documentos_adjuntos.all()

# Check which rendiciones use a document
documento = Documento.objects.get(id=5)
rendiciones = documento.rendiciones_adjuntas.all()
```

## Upload-First Architecture

Auditoriapp follows an "upload-first" pattern:

<Steps>
  <Step title="Upload Document">
    User uploads file via the documents endpoint
    ```python
    POST /api/documentos/
    # Returns document ID
    ```
  </Step>
  
  <Step title="Get Document ID">
    Frontend receives the created document's ID
    ```json
    {
      "id": 123,
      "nombre": "acta_asamblea.pdf",
      "tipo": "acta"
    }
    ```
  </Step>
  
  <Step title="Associate with Project">
    When creating/updating project, reference the document ID
    ```python
    PATCH /api/proyectos/1/
    # Link document via proyecto field or ManyToMany
    ```
  </Step>
</Steps>

<Warning>
**Why Upload-First?** This pattern ensures files are safely stored before being referenced, preventing orphaned database records and improving error handling.
</Warning>

Physical progress reports can include photos:

```python
class ReporteAvance(models.Model):
    proyecto = models.ForeignKey(Proyecto, on_delete=models.CASCADE, 
                                 related_name='reportes_avance')

                              null=True)
```

## API Endpoints

### Upload Document


**Request:**

**Response:**

```json
  "fecha_subida": "2026-03-14T10:30:00Z",
  "proyecto": 1,
  "archivo": "/media/documentos/acta_asamblea_marzo.pdf",
  "descripcion": "Aprobación proyecto mejoramiento plaza"

```

```http
GET /api/documentos/?proyecto=1
```

```http
GET /media/documentos/acta_asamblea_marzo.pdf
```

<Note>
Media files are served through Django's development server in DEBUG mode. For production, configure a proper file server (nginx, S3, etc.).
</Note>

## Multi-Tenant Document Isolation

Documents must respect community boundaries:

```python
class DocumentoViewSet(viewsets.ModelViewSet):
    serializer_class = DocumentoSerializer
    permission_classes = [IsAuthenticated]
    
    def get_queryset(self):
        user = self.request.user
        if hasattr(user, 'comunidad') and user.comunidad:
            # Only return documents for projects in user's community
            return Documento.objects.filter(
                Q(proyecto__comunidad=user.comunidad) | 
                Q(proyecto__isnull=True, comunidad=user.comunidad)
            )
        return Documento.objects.none()
```

<Warning>
**Security Critical**: Always filter documents by community to prevent cross-tenant data access.
</Warning>

## Document Lifecycle

### Typical Document Flow

<Steps>
  <Step title="Upload">
    User uploads document through the API
  </Step>
  <Step title="Review">
    CPA/Admin reviews document during project/rendicion approval
  </Step>
  <Step title="Archive">
    Document remains accessible for audit purposes
  </Step>
  <Step title="Retention">
    Documents retained per legal requirements (typically 5-10 years)
  </Step>
</Steps>

### Deletion Policy

<Warning>
**Important**: Documents should generally NOT be deleted once associated with approved projects or expense reports due to audit requirements.
</Warning>

```python
# Soft delete implementation
class Documento(models.Model):
    # ... other fields
    activo = models.BooleanField(default=True)
    fecha_eliminacion = models.DateTimeField(null=True, blank=True)
    
    def soft_delete(self):
        self.activo = False
        self.fecha_eliminacion = timezone.now()
        self.save()
```
